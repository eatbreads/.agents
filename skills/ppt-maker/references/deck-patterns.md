# Deck Patterns

Use these patterns when the user gives only a broad topic or asks for help structuring a presentation.

## Status Update

1. Title / purpose
2. Executive summary
3. Background or target
4. Progress snapshot
5. Key evidence or metrics
6. Risks and blockers
7. Next steps
8. Ask / decision needed

## Proposal

1. Title / proposal statement
2. Problem
3. Why now
4. Options considered
5. Recommended option
6. Expected impact
7. Implementation plan
8. Risks and mitigation
9. Decision request

## Technical Sharing

1. Topic and takeaway
2. Context / problem
3. System or architecture view
4. Core mechanism
5. Key implementation details
6. Tradeoffs
7. Results / lessons learned
8. Q&A

## Training

1. Learning objective
2. Why this matters
3. Concept 1
4. Concept 2
5. Worked example
6. Common mistakes
7. Practice / checklist
8. Summary

## Per-Slide Checklist

- Does the title express the conclusion, not just the topic?
- Can the audience understand the point in under five seconds?
- Is there one message instead of several competing messages?
- Does the slide need a visual instead of more bullets?
- If this claim matters, is there evidence on the slide or in notes?
